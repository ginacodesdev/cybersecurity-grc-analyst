## What Is NIST SP 800-53

NIST Special Publication 800-53 is the catalog of
security and privacy controls for federal information
systems and organizations. It is one of the most
comprehensive and authoritative control catalogs
in existence and serves as the primary reference
for control selection in the Risk Management
Framework. While it was originally developed
for federal agencies it is widely adopted in
the private sector as a baseline for building
robust security programs.

The current version, Revision 5, was released
in 2020 and introduced significant changes including
the integration of privacy controls alongside
security controls, expanded supply chain risk
management guidance, and a shift toward outcome-based
control statements that apply across different
types of computing environments.

## How SP 800-53 Is Organized

SP 800-53 organizes its controls into families.
Each family addresses a specific domain of
security or privacy and contains multiple
individual controls with associated control
enhancements. The organization of families
makes it possible to focus on specific areas
of the security program when conducting gap
analyses or planning remediation efforts.

| Family | Abbreviation |
|--------|-------------|
| Access Control | AC |
| Awareness and Training | AT |
| Audit and Accountability | AU |
| Assessment, Authorization, and Monitoring | CA |
| Configuration Management | CM |
| Contingency Planning | CP |
| Identification and Authentication | IA |
| Incident Response | IR |
| Maintenance | MA |
| Media Protection | MP |
| Physical and Environmental Protection | PE |
| Planning | PL |
| Program Management | PM |
| Personnel Security | PS |
| PII Processing and Transparency | PT |
| Risk Assessment | RA |
| System and Services Acquisition | SA |
| System and Communications Protection | SC |
| System and Information Integrity | SI |
| Supply Chain Risk Management | SR |

## Control Baselines

SP 800-53 defines three control baselines that
correspond to the impact levels established during
the categorization step of the RMF. Each baseline
is a starting point for control selection and
can be tailored to fit the specific needs of
the system being secured.

| Baseline | Impact Level | Scope |
|----------|-------------|-------|
| Low | Low impact systems | Minimum set of controls for systems where compromise would have limited effects |
| Moderate | Moderate impact systems | Expanded control set for systems with more significant risk |
| High | High impact systems | Comprehensive control set for systems where compromise would have catastrophic effects |

## SP 800-53 and the GRC Analyst

SP 800-53 is one of the most frequently referenced
documents in federal and defense GRC work. A GRC
Analyst working in these environments needs to
be comfortable navigating the catalog, understanding
what individual controls require, and translating
those requirements into actionable guidance for
the system teams responsible for implementation.

The RA family is particularly relevant to daily
GRC work as it covers risk assessments, vulnerability
monitoring, and threat intelligence activities.
The SI family governs flaw remediation and system
monitoring which are central to vulnerability
management and POAM activities. The CA family
covers the assessment, authorization, and continuous
monitoring activities that define the RMF process.

---

## SP 800-53A — The Assessment Procedures Companion

NIST SP 800-53A is the companion document to
SP 800-53. While SP 800-53 defines what controls
are required, SP 800-53A provides the assessment
procedures that describe how to determine whether
each control has been implemented correctly and
is operating effectively. Assessors use SP 800-53A
as the primary guide for conducting security
control assessments during Step 5 of the RMF.

A GRC Analyst who understands both SP 800-53 and
SP 800-53A has a complete picture of what is
required and how it will be evaluated, which
is exactly the perspective needed to prepare
a system for a successful assessment.

## Key Terms

| Term | Definition |
|------|-----------|
| Control Family | A grouping of related security or privacy controls that address a common area |
| Control Enhancement | An addition to a base control that provides increased security capability |
| Baseline | A pre-defined set of controls mapped to a specific system impact level |
| Tailoring | The process of adjusting a control baseline to fit the specific context of a system |
| SP 800-53A | The NIST assessment procedures companion to SP 800-53 |
| Overlay | A set of tailoring actions applied to a baseline for a specific technology or community of interest |
